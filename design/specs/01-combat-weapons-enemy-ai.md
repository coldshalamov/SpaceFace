# Combat, weapons & enemy AI

## Summary
A fixed-60Hz, data-driven combat layer resolving all damage through a unified shield -> armor -> hull pipeline with 4 damage types (kinetic, energy, explosive, thermal), one shared energy capacitor powering weapons + afterburner boost, and 6 data-defined weapon archetypes (pulse, beam, autocannon, missile, flak/PD, mining-laser-as-weapon). Ships expose typed hardpoints (S/M/L) + module slots; player and NPCs share one `combatants` array and one pooled `projectiles` array so the same resolver, targeting, and steering run for both. Enemy AI is a steering + finite-state-machine stack across 6 archetypes (swarmer, sniper, brawler, fleeing trader, pirate, mini-boss capital), driven by an aggro/threat table. Enemy strength, loot, and credit rewards scale off sector.securityLevel (0..5 danger tier). Player death runs an insurance/respawn flow that refunds a fraction of hull+cargo value and returns to last docked station. All state lives in shared GameState; systems talk via the event bus (combat:damage, combat:kill, combat:fire, ai:stateChange, player:death, loot:drop) so trade/mining/economy/save compose without combat knowing their internals.

## Mechanics
- DEFENSE LAYERS: every combatant has shield (regenerating energy buffer), armor (flat-mitigation plate, slow/no regen), hull (HP; 0 = destroyed). Incoming damage hits shield first, overflow to armor+hull. While shield > 0, armor's flat reduction does NOT apply (shield absorbs raw typed damage); once shield depletes, remaining damage in that same hit spills to hull with armor flat reduction applied.
- SHIELD REGEN: shield does not regen for `shieldRegenDelay` seconds after the last damage taken (default 4.0s; resets on every hit). After delay, regen at `shieldRegenRate` shield/s (default 8% of shieldMax per s). Shield regen draws 0 capacitor (separate shield emitter). A 'shieldDown' event fires when shield hits 0; 'shieldRestored' when it returns above 25% max.
- ARMOR: flat reduction `armorFlat` subtracted per hit AFTER damage-type multiplier, floored so each hit always deals >= 1 (or >= 10% of post-type damage, whichever larger) to prevent immunity. Armor has its own HP pool `armorMax`; damage to hull first chews `armorHp` (with flat reduction) then hull. Armor regens only at stations/repair, never in-combat. Kinetic gets -50% armorFlat effectiveness (AP rounds), explosive ignores 30% of armorFlat.
- DAMAGE TYPES & MULTIPLIERS vs [shield, armor, hull]: kinetic [0.8, 1.0, 1.0]; energy [1.25, 0.7, 1.0]; explosive [0.9, 1.15, 1.3]; thermal [1.1, 1.1, 0.9]. Lasers=energy (shred shields), autocannons=kinetic (balanced, pierce armor), missiles/torpedoes=explosive (wreck hull/armor), mining laser=thermal (slow, anti-rock). This makes weapon mixing meaningful.
- ENERGY CAPACITOR: single pool `capMax` (cr-bought module), regen `capRegen`/s. Every weapon shot subtracts `energyCost`; if cap < cost the weapon cannot fire (click/empty SFX, no shot). Afterburner boost drains `boostDrain`/s while held and gives +60% max speed + faster accel; releasing boost or hitting 0 cap ends it. Beam laser drains continuously while firing (cost/s). Shield emitter regen is OFF the capacitor (so you can't fully brick yourself).
- HEAT (lightweight): weapons accumulate `heat` per shot; per-weapon `heatMax`. At heatMax the weapon overheats and locks out for `cooldownLockS` (default 2.0s, audible). Heat dissipates at `heatDissip`/s when not firing. Most weapons are energy-gated not heat-gated; autocannon and beam are heat-gated to cap sustained DPS. Heat is per-weapon-instance, not shared.
- HARDPOINTS & SLOTS: each ship hull defines hardpoints by size class S/M/L and an arc (fixed-forward, or turret with `turretArcDeg`). A weapon of size <= hardpoint size fits (smaller weapon in bigger slot allowed, wastes potential). Separate `moduleSlots` (defensive/utility: shield booster, capacitor, armor plate, afterburner, scanner, drone bay). Player ship config stored as arrays of mounted item ids referencing the data tables.
- FIRING: weapons fire on a per-weapon cooldown = 1/rof. `groupFire` lets player bind weapons to fire-group 1/2 (e.g. all guns on LMB, missiles on RMB). On fire: check cooldown + energy + heat + (for missiles) ammo + lock; spawn projectile(s) from hardpoint world position with muzzle velocity = ship velocity + forward*projectileSpeed (inherit momentum). Beam/PD are hitscan-ish (instant ray each tick while active).
- TARGETING & LOCK: player presses Target-Nearest (Tab) or Target-Under-Reticle to set `state.player.targetId`. Lock-on for missiles requires holding target in a `lockConeDeg` (default 25 deg) for `lockTimeS` (default 1.2s, faster for smaller missiles); lock decays if target leaves cone. Locked target gets a missile. Hardpoint turrets auto-aim at current target within their arc.
- LEAD INDICATOR: for the current target and the player's primary projectile weapon, compute intercept point (iterative, 2 passes) from relative position + relative velocity + projectileSpeed; HUD draws a lead pip there. Hitting the pip = hit. Beam/PD show no lead (instant). AI snipers use the same intercept math to aim.
- PROJECTILE LIFECYCLE & POOLING: fixed pre-allocated pools per projectile category (bullets 512, missiles 64, beams handled as transient rays not pooled objects). Spawn = take inactive slot, set pos/vel/owner/damage/type/lifeS; if pool exhausted, recycle oldest. Each tick: integrate pos += vel*dt, decrement lifeS, broad-phase grid collision vs combatants (skip owner + same-faction unless friendly-fire on). On hit or lifeS<=0: emit damage, deactivate, return to pool. Missiles additionally steer toward target each tick (tracking).
- COLLISION: uniform spatial hash grid (cell ~120 wu) holds combatant bounding circles; projectiles query only their cell + 8 neighbors. Hit test = point/segment vs circle (radius = ship `collisionRadius`). Continuous (swept segment) test for fast bullets to avoid tunneling. Flak/PD also test vs enemy projectiles (intercept incoming missiles).
- ENEMY AI STACK: each NPC has (1) steering layer producing a desired acceleration from blended behaviors (seek/flee/arrive/pursue/evade/separation/wander/orbit/strafe), clamped to ship accel/turn limits; (2) FSM choosing which behaviors are active + weapon usage (idle, patrol, pursue, attack, strafe, flee, dead). Sensors: target acquisition by range+faction hostility+threat table.
- AGGRO/THREAT: each NPC keeps a small threat map {attackerId: threat}. Dealing damage adds threat = damageDealt; being the nearest hostile adds proximity threat. NPC targets highest-threat in sensor range. Player actions (firing on faction, stealing kills) raise faction-level reputation handled by economy/faction system; combat just emits events and reads `state.factions[x].hostileTo`.
- DIFFICULTY SCALING: spawner (separate system) reads sector.securityLevel; this subsystem provides `scaleCombatant(baseEnemy, dangerTier)` that multiplies hull/shield/damage and bumps weapon tier per the scaling formula. Mini-boss spawns gated to low-security/special sectors.
- LOOT & CREDITS: on combat:kill of an NPC, roll loot from the enemy's `lootTable` (credits range, commodity drops, rare module/weapon chance) scaled by dangerTier; spawn pickup entities (handled by mining/pickup system via loot:drop event) or grant credits directly. Bounty credits granted instantly to player on kill.
- PLAYER DEATH/RESPAWN: when player hull <= 0, emit player:death, freeze input, play destruction, then offer insurance respawn: refund = insuranceRate * (hullRebuildCost + insuredModulesValue) - deductible; lose `cargoLossRate` of cargo; respawn at last docked station with hull/shield full after `respawnDelayS`. If credits+insurance can't cover, fallback to a free basic 'loaner' starter ship (no game-over, idle-growth friendly).

## State Owned
- combatants: Combatant[] — unified array of all active ships (player + NPCs). Each: {id, factionId, isPlayer, pos:{x,z}, vel:{x,z}, heading(rad), hull, hullMax, armorHp, armorMax, armorFlat, shield, shieldMax, shieldRegenRate, shieldRegenDelay, lastDamageT, cap, capMax, capRegen, collisionRadius, mass(t), maxSpeed, accel, turnRate(rad/s), hardpoints:Hardpoint[], modules:string[], targetId, lockTarget, lockProgress, ai:AIState|null, lootTableId, bountyCr, heatByHardpoint:number[]}
- projectiles: Projectile[] — pooled. Each: {active:bool, kind:'bullet'|'missile', ownerId, factionId, pos, vel, dmg, dmgType, lifeS, radius, targetId(missiles), turnRate(missiles), armed(missiles)}
- beams: Beam[] — transient per-tick active beam/PD rays {ownerId, factionId, from, to, dmgType, dpsRemainingThisTick} (rebuilt each tick, not persisted)
- player.targetId: string|null — current selected target
- player.fireGroups: {1:hardpointIdx[], 2:hardpointIdx[]} — input binding of hardpoints to fire buttons
- player.boostActive: bool — afterburner currently engaged
- player.insurance: {rate:0..1, deductibleCr, insuredModules:bool, lastStationId} — respawn policy
- spatialGrid: transient uniform-hash {cellSize, map<cellKey, combatantId[]>} rebuilt each tick (not serialized)
- combatRng: seeded PRNG state (uint32) — deterministic loot/spread rolls, serialized for determinism
- sector.securityLevel: int 0..5 (READ; owned by sector/world system) -> dangerTier = 5 - securityLevel

## Formulas
- dt = 1/60 s (fixed sim step). All rates (regen, DPS, drain, heat) multiply by dt when applied per tick.
- DAMAGE RESOLUTION (single hit of raw `D` and type `T` against combatant C):  let mS=typeMult[T].shield, mA=typeMult[T].armor, mH=typeMult[T].hull.  IF C.shield > 0:  dShield = D * mS;  absorbed = min(C.shield, dShield);  C.shield -= absorbed;  leftoverRaw = (dShield - absorbed) / mS  (raw damage not absorbed, converted back);  D = leftoverRaw  (continues to armor/hull only if shield broke this hit).  IF D > 0 (shield gone):  dBody = D * mH;  // apply armor flat reduction with type modifiers:  effFlat = C.armorFlat * (T=='kinetic'?0.5 : T=='explosive'?0.7 : 1.0);  dAfterArmor = max(dBody - effFlat, max(1, 0.1*dBody));  // chew armorHp first:  toArmor = min(C.armorHp, dAfterArmor * mA);  C.armorHp -= toArmor;  remainder = dAfterArmor - toArmor/mA;  C.hull -= max(0, remainder * mH... )  -> simplified impl: if armorHp>0 subtract from armorHp (scaled mA) else from hull (scaled mH).  Set C.lastDamageT = now. emit combat:damage{targetId,attackerId,amount,type,brokeShield}.  If C.hull<=0 -> destroy.
- SHIELD REGEN (per tick, if now - lastDamageT >= shieldRegenDelay AND shield < shieldMax): shield = min(shieldMax, shield + shieldRegenRate*dt). (shieldRegenRate stored as absolute shield/s; default = 0.08*shieldMax for ~12.5s full recharge.)
- CAP REGEN (per tick): cap = min(capMax, cap + capRegen*dt). WEAPON FIRE GATE: canFire = (cap >= energyCost) && (heat + heatPerShot <= heatMax) && (cooldownTimer<=0) && (ammoOk) && (lockOk for missiles). On fire: cap -= energyCost; heat += heatPerShot; cooldownTimer = 1/rof. BOOST: while boostActive && cap>0: cap -= boostDrain*dt; maxSpeedEff = maxSpeed*1.6; accelEff = accel*1.5. else maxSpeedEff=maxSpeed.
- HEAT (per weapon per tick when NOT firing): heat = max(0, heat - heatDissip*dt). If heat >= heatMax: overheated=true, lockTimer=cooldownLockS; can't fire until heat<heatMax*0.5 OR lockTimer elapsed.
- PROJECTILE INTEGRATE (per active bullet/missile per tick): pos += vel*dt; lifeS -= dt; if lifeS<=0 deactivate. MISSILE TRACKING: desired = normalize(target.pos - pos); cur = normalize(vel); newDir = rotateToward(cur, desired, turnRate*dt); speed=min(maxProjSpeed, |vel| + accel*dt); vel = newDir*speed.
- LEAD/INTERCEPT (target T, shooter S, projectile speed p): rel = T.pos - S.pos; relV = T.vel - S.vel; solve |rel + relV*t| = p*t  -> quadratic a=relV·relV - p^2, b=2*rel·relV, c=rel·rel; t = smallest positive root; aimPoint = T.pos + relV*t. (Iterate 2x for moving shooter.) If no positive root (can't catch), aim directly at target.
- SPREAD: fireDir = rotate(baseDir, gaussian(0, spreadDeg)). Deterministic via combatRng.
- DANGER TIER: dangerTier = clamp(5 - sector.securityLevel, 0, 5).
- ENEMY SCALING (scaleCombatant(base, tier)): hullMax *= (1 + 0.25*tier); shieldMax *= (1 + 0.30*tier); armorFlat *= (1 + 0.20*tier); weaponDmg *= (1 + 0.18*tier); bounty *= (1 + 0.5*tier); maxSpeed *= (1 + 0.04*tier). Round to ints. Apply at spawn only.
- AGGRO TARGET SELECT: score(candidate) = threat[candidate.id] + 50/(1+dist/100) + (candidate.isPlayer?20:0); pick max within sensorRange & hostile faction. threat[attacker] += damageDealt on each combat:damage where this NPC is target; threat decays *0.98/s.
- STEERING BLEND: accel = clamp( w_seek*seek + w_sep*separation + w_evade*evade + w_orbit*orbit, maxAccel ); heading turns toward accel dir at turnRate*dt. Per-archetype weights (e.g. swarmer: orbit 1.0, separation 0.8, evade 0.5).
- FLEE/THREAT-DECIDE: if hull/hullMax < fleeThreshold (pirate 0.2, trader always flees) -> FSM->flee. if target out of sensorRange for >3s -> ->patrol.
- LOOT ROLL: credits = randInt(lootMin, lootMax) * (1 + 0.5*dangerTier); per drop entry roll chance*(1+0.1*dangerTier); use combatRng for determinism.
- BOUNTY PAYOUT: on kill of bountied NPC -> state.player.credits += round(bountyCr * (1+0.5*dangerTier)); emit economy:credit{amount,reason:'bounty'}.
- INSURANCE REFUND on death: refund = clamp( insurance.rate * (shipBaseValue + (insurance.insuredModules ? modulesValue : 0)) - insurance.deductible, 0, +inf ); cargoKept = cargo * (1 - cargoLossRate). If player.credits + refund < loanerThreshold -> grant free Shuttle loaner.

## Interactions
- EMITS combat:fire {ownerId, weaponId, hardpointIdx, origin, dir} -> audio system plays procedural weapon SFX; vfx system spawns muzzle flash.
- EMITS combat:projectileSpawn {projId, kind} and combat:projectileHit {projId, targetId, pos} -> vfx (impact sparks/explosions), audio.
- EMITS combat:damage {targetId, attackerId, amount, type, brokeShield, isPlayer} -> HUD (damage numbers, shake), audio; AI threat update; reputation/faction system (if attackerId==player & target is lawful faction -> wanted++).
- EMITS combat:kill {victimId, killerId, factionId, bountyCr, lootTableId, pos} -> loot system spawns pickups; economy grants bounty; mission system checks kill objectives; stats/achievement counters; faction reputation delta.
- EMITS loot:drop {pos, credits, items[]} -> pickup/mining system materializes collectible entities (reuses mining pickup pipeline).
- EMITS ai:stateChange {npcId, from, to} -> debug overlay; optional audio bark; minimap threat icons.
- EMITS player:death {pos, killerId} -> input system locks controls; UI shows respawn screen; save system optional autosave; music system stinger.
- EMITS player:respawn {stationId, shipId, refundCr, cargoLost} -> world teleports player, HUD resets, economy applies refund/deductible.
- EMITS shieldDown {combatantId} / shieldRestored {combatantId} -> HUD + audio.
- READS state.sector.securityLevel + spawn tables (owned by world/spawner system) to scale enemies via scaleCombatant.
- READS state.player.credits, .cargo, .ship config (owned by economy/inventory) for ammo consumption (missiles/torpedoes decrement cargo commodity), insurance payout, mining-laser ammo-free check.
- READS state.factions[*].hostileTo / reputation (owned by faction system) to decide friendly-fire and AI hostility; combat never mutates rep directly, only emits events.
- WRITES state.player.credits via economy:credit event (bounty/loot) rather than direct mutation, to keep economy authoritative; direct mutation allowed only as fallback if economy unsubscribed.
- CALLS mining system contract: mining laser emits mining:hit {asteroidId, dmg, type:'thermal'} so the mining system (not combat) handles ore yield; combat only applies the beam ray + heat/energy.
- SAVE/LOAD: combatants(non-player NPCs are regenerated by spawner, NOT serialized), player combat fields (hull/shield/cap/modules/hardpoints/insurance/targetId) ARE serialized; projectiles & beams & spatialGrid are transient (cleared on load); combatRng seed serialized for determinism.
- UPDATE ORDER (within fixed tick): 1 input->intents, 2 AI think (FSM+steering+target), 3 movement integrate (apply accel/turn/boost), 4 weapons fire (spawn projectiles/beams, spend cap/heat), 5 projectile integrate + tracking, 6 rebuild spatial grid, 7 collision + damage resolution, 8 shield/cap/heat regen, 9 death checks + loot/respawn, 10 emit HUD/vfx events. Combat system slots after input & before render.

## UI Needs
- HUD ship status: player hull bar (HP + %), armor bar, shield bar (with regen-delay shimmer/countdown), capacitor bar (with low-cap warning flash), per-weapon heat bars + overheat lockout indicator.
- Weapon group readout: which weapons in group 1/2, ready/cooldown/overheat/out-of-ammo/no-lock state per hardpoint, missile ammo count.
- Target panel: selected target name/faction, hull/armor/shield bars, distance (wu), relative speed, threat/hostility color, subsystem health for mini-boss (per-turret HP).
- Lead indicator: floating pip at computed intercept point for current target + primary weapon; in-range/out-of-range color; turns red when target leaves optimal range.
- Lock-on reticle: animated lock progress ring around target while acquiring missile lock; solid when locked; shows lock cone hint.
- Incoming-missile warning: directional arrow + audio when a tracking projectile targets player; PD-active indicator.
- Aggro/threat hint: minimap/radar shows hostile blips, count of active hostiles, 'pack incoming' alert; mini-boss phase-transition banner.
- Damage feedback: floating damage numbers (color by type/whether shield was hit), screen-edge directional damage indicator, screen shake on big hits, low-hull red vignette.
- Kill/loot toast: '+X cr bounty', loot items collected, kill streak; XP/progression hook if leveling exists.
- Death/respawn screen: cause of death, insurance breakdown (ship value, refund, deductible, cargo lost), 'Respawn at [station]' button + countdown, loaner-ship notice if broke.
- Boost/afterburner indicator: active state + cap drain rate; speed readout (wu/s).
- Loadout/shipyard UI hooks (mounted at station, owned by shipyard system but fed by this system's data): hardpoint size/arc per slot, weapon stat tooltips (DPS, energy/s, range, DPS-per-cap, dmg-type vs shield/armor/hull), module slots, fit validation (size fits, total cap drain vs regen warning).
